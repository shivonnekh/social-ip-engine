014 Macro Phases

Every patient conversation goes through four phases. Product appears in Phase 3, clinic booking in Phase 4 — both gated by deterministic preconditions plus LLM judgment of patient readiness.

Phase 1

Greeting · 招呼

~T1
Doctor portrait + Jessica intro.
Tools: send_doctor_greeting
Phase 2

Diagnosis · 診斷

~T2 → T13
探病情 → 望脷象 → 5-Q 體質問卷 → 宣布體質.
Tools: record_complaint, record_complaint_probe (×3), request_tongue_photo, analyze_tongue, describe_tongue_findings, ask_constitution_question (×5), record_q_answer (×5), declare_constitution, share_diagnosis
Phase 3

Product · 產品

~T15 → T16 (after 1-2 caring turns)
Maca 極致黑 + product photo. Earliest possible firing.
Tools: share_product
Phase 4

Clinic · 預約

~T17 → T20
District match → propose slots → confirm.
Tools: match_clinic, propose_booking_slot, confirm_booking

04Hard Gates — When Tools Reject

The LLM has agency over when to call tools, but every tool checks preconditions before mutating state. These hard gates prevent rushed pitches and hallucinated diagnoses.

declare_constitution — 5/5 questions required

Tool returns error: missing_quiz_answers if any of Q1-Q5 unanswered. Listed missing keys + count returned. Even when pacing_compress=true (patient seems fatigued), no bypass — the agent must pivot to clinic invite instead.

Fixed: src/tools/sales.py:2088-2102 · Tests: test_declare_constitution_needs_both_answers

share_product — requires diagnosis_shared=True

Cannot pitch product before diagnosis. Check at src/tools/sales.py: if not sales_state.get("diagnosis_shared"): → returns error: diagnosis_not_shared_yet. This is what enforces the 1-2 caring turn buffer between Phase 2 and Phase 3.

match_clinic — requires product_pitched=True

Cannot start booking flow before product. Check: if not sales_state.get("product_pitched"): → returns error: no_product_pitched_yet. Forces the macro-phase order: 診斷 → 產品 → 預約.

propose_booking_slot — requires matched_clinic_id

Cannot propose slots before clinic is matched. Returns error: no_clinic_matched_yet. The slot proposer reads opening hours from the matched clinic, not a default.

analyze_tongue — must run before describe_tongue_findings

The describe tool requires tongue_findings in state (set by analyze_tongue). Returns error: analyze_tongue_must_run_first. Prevents LLM from authoring tongue readings without vision input.

Pacing — no clinic / product mention before Turn 4

Soft rule (prompt-enforced, not tool-gated): src/sales/sales_prompt.py:200-205 — "唔可以喺 Turn 4 之前提診所/見醫師/預約". Lets the LLM build trust before pivoting.

05LLM Agency vs Deterministic Gates

Where the LLM decides vs where the system enforces. Critical to understand: even though the funnel feels deterministic, every tool call is an LLM choice.

🧠 LLM Decides

  • When to call record_complaint (after patient describes symptom)
  • Which probes to ask (timing / severity / triggers / past)
  • Which question phrasing for empathy
  • How long to wait between diagnosis and product (1 vs 2 caring turns)
  • Whether to fire share_product yet (patient ready? still asking?)
  • How to compose the diagnosis Cantonese narrative (using YAML recommendations as source-of-truth)
  • When to call match_clinic (after product fires + patient interest signal)

🔒 System Enforces

  • Q1-Q5 all answered before declare_constitution succeeds
  • diagnosis_shared must be true before share_product
  • product_pitched must be true before match_clinic
  • matched_clinic_id required for propose_booking_slot
  • Constitution name must appear in diagnosis_text (post-validation)
  • SOS pattern matching overrides everything (胸痛, 想死, etc.)
  • Acupoint anti-repeat ledger blocks already-mentioned points
