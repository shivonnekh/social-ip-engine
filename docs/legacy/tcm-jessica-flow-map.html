039 Stages of the Journey

Each turn the LLM reads sales_state from the prompt, identifies the current stage, and calls the matching tool(s). Stages advance only when their tool's preconditions are met — every tool has a hard guard.

0Intro / greeting

Trigger: sales_state empty OR only greeting_shown=True (stale flag)

send_doctor_greeting

Queue lead bubble (canonical greeting text) + media (Jessica portrait /static/sales/hong_doctor.jpg). Set greeting_shown=True.

→ state: + greeting_shown

1Complaint capture

Trigger: user shares 主訴 (失眠 / 頭痛 / 經痛 / 美容 / etc.)

record_complaint request_tongue_photo

Both fire in the same turn. Reply combines: empathy ack + 1 clarifying Q + 望聞問切 framing + tongue ask — all in a single message (split by \n).

→ state: + complaint, complaint_recorded_at, tongue_requested

2-3Tongue analysis

Trigger: user replies with photo (or text deferral)

analyze_tongue describe_tongue_findings (optional)

3-tier vision cascade. Three outcomes:

  • Real tongue → real findings → tongue_described=True
  • Vision API down → fake generic reading → tongue_described=True (journey continues)
  • NOT a tongue → polite redirect → tongue_described=False (journey paused, retry expected)

→ state: + tongue_findings, tongue_described, tongue_description_text

4Constitution Q1 — energy

Trigger: tongue step done OR photo deferred

ask_constitution_question(1)

Queues scripted bubble verbatim (LLM never writes A/B/C — prevents character drift like 攰→劫). Reply text = short transition (≤20 chars) like "等我再問你兩條題 😊".

→ state: + q1_asked

5Q1 answer + Q2 — temperature

Trigger: user sends single letter A/B/C (or "A就啱")

record_q_answer(1, letter) ask_constitution_question(2)

Both fire in the same turn. Q2 turn LLM reply text is wholesale-suppressed by post-processor — Q2 scripted bubble lands clean with no redundant intro.

→ state: + constitution_answers.q1, q2_asked

6Constitution declaration + product pitch

Trigger: Q2 answered (Q1 + Q2 both in constitution_answers)

record_q_answer(2, letter) declare_constitution share_diagnosis

3 tools in 1 turn. Reply is 3 paragraphs separated by \n\n:

  • Part A — diagnosis ("根據你嘅脷相 + 答案，你可能主要屬於 陽虛質 🌿 ...")
  • Part B — Maca product pitch with full URL ("呢個體質啱用 Dr. Jessica 極致黑G瑪卡 ✨ 👉 [URL]")
  • Part C — soft-close ("如果想更精準調理, 可以嚟我哋 clinic 見醫師面診 🌿")

+ Maca product photo attachment.

→ state: + dominant_constitution, constitution_distribution, diagnosis_shared

7Clinic match

Trigger: user expresses booking interest + provides specific district

match_clinic(district)

Hard rejects regional umbrella terms — "九龍" / "Kowloon" / "港島" / "新界" all return district_too_broad; LLM must ask for specific district. Self-contained lead bubble (clinic name + address + travel hint + "幾時過嚟?" question). LLM reply suppressed (clinic card stands alone).

→ state: + user_district, matched_clinic_id

8Booking slot proposal

Trigger: user gives date/time hint ("聽日下午" / "星期六上午")

propose_booking_slot(date_hint, time_hint)

Walks forward 14 days from HKT today (not server UTC) — anchored to fix the timezone-edge bug where 6am HKT showed yesterday's "tomorrow". Picks first open slot in clinic's opening_hours.

→ state: + proposed_booking_date, proposed_booking_time

9Booking confirmation

Trigger: user confirms slot ("好" / "OK" / "嗯" / "可以" / "👍")

confirm_booking

Returns booking_full_display_zh ("2026年4月29日（星期三）下午3:30") which the LLM uses verbatim in reply (LLM has hallucinated dates like "2024年2024" when reformatting raw ISO).

→ state: + booking_confirmed=True, journey_complete=True, booking_confirmed_at (HKT-aware)

🎉 Journey complete → falls through to FAQ mode

07Side Flows

Patterns that branch off the main funnel. Each handles a real conversational pattern observed in production.

A. Photo deferral

Trigger: "等下影" / "OK" / "知喇" / "冇相" / "影唔到"

LLM calls ask_constitution_question(1) directly (no tongue findings) → Q1 fires; tongue is bonus, never blocker. State: tongue_requested=True, tongue_findings={}, tongue_described=False.

B. Mid-journey FAQ

Trigger: "幾錢?" / "點影?" / 穴位 questions

LLM calls find_knowledge_cards(query) → answers from Qdrant cards → soft-bring-back to next journey step. No state change.

C. Refusal / escape

Trigger: "唔 book 啦" / "等我諗下"

LLM acknowledges warmly, journey pauses; subsequent messages auto-route to FAQ mode (still Jessica prompt, no push).

D. Reactive constitution probe

Trigger: 調理 / 補身 / 點改善 + low constitution signal

Dispatcher returns Decision(ASK | RETRY | RESUME | DROP | PASSTHROUGH). ASK injects diagnostic Q before answering original; RESUME merges score then answers; DROP apologises briefly.

E. Wait mode (tongue pending)

Trigger: tongue requested but not described, user sends non-photo content

Short ack, NO new question, gentle remind to send tongue. Photo deferral still wins if user explicitly defers.

F. NOT-a-tongue retry

Trigger: vision providers reject image as non-tongue

Lead bubble = polite redirect. tongue_described=False. tongue_invalid_attempt_count++. User can retry with a real tongue — idempotency guard doesn't block (no real findings yet).

08Proactive Triggers (Background, Scheduled)

Fire AFTER each turn. SQLite-backed scheduler polls every 60s, respects HK quiet hours 22:00–08:00, sends via ChatDaddy. Cancel-on-progress sweep removes stale follow-ups when user advances.

Detection-driven

TriggerDelayDetection signalCancel rule
emotional_checkin 24h regex: 擔心 / 好辛苦 / 壓力大 / 焦慮 / stressed / depressed Cancel if user signals positive affect
symptom_followup 48h regex: 失眠 / 頭痛 / 經痛 / 胃痛 / 便秘 / 暗瘡 (canonical name captured) Cancel if specific symptom resolved

Funnel-driven (sales-state stage)

TriggerDelayWhen firesCancel rule
funnel_complaint_pending 24h greeting_shown=True, no complaint Cancels when complaint set
funnel_tongue_pending 24h complaint set, no tongue ack Cancels when tongue described
funnel_q1_pending 24h tongue done, no Q1 Cancels when Q1 answered
funnel_q2_pending 24h Q1 done, no Q2 Cancels when Q2 answered
funnel_booking_pending 48h diagnosis shared, no clinic match Cancels when clinic matched

Disabled by design

  • welcome_checkin — intrusive for no-progress user
  • constitution_journey 9-Q over 27 days — Phase 0 (data + scoring exist), scheduler not wired (see docs/plans/constitution-journey-proactive-wiring.md)
