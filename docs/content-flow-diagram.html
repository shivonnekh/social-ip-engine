Safety · shipped 2026-07-03

Bounce the stage all you like — it never re-runs

Three independent guards + everything persisted to git, so nothing vanishes on deploy.

🛡️

Row guard

Processed rows recorded in a git-saved state file. Re-flips are skipped instantly.

main guard
🔑

Keyword guard

A rule for the same keyword already exists? Skipped. No duplicates, ever.

backup guard
💰

Image guard

Generated PNGs are reused, never re-generated. No double spend on the image API.

cost guard
🔒
A row only re-wires if you deliberately delete its rule or change its keyword — never by accident.
