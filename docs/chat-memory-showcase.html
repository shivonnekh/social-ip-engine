觸發條件：用戶有過往對話歷史 + 訊息含「拜拜 / 再見 / bye / 多謝 / thx / 好啦 / 晚安」等告別詞

3. 長期記憶（跨 session 自動摘要）

每 15 條新對話，Jessica 自動寫一份「記憶摘要」入用戶 CRM。 即使用戶傾咗 100 條訊息嘅內容，重要嘢都唔會漏。

觸發條件：後台自動運行 — 用戶累積 15+ 條新訊息後，Haiku model 默默 update CRM notes（零延遲，不影響回覆速度）

📝 自動產生嘅 CRM 記憶範例：

用戶：Apple（+852 9xxx xxxx）
體質：陽虛質
記憶摘要：
• 護士工作，夜更多，長期睡眠不足
• 主要困擾：失眠（連續 3 個月）、腰部僵硬
• 飲過蓮子百合湯，覺得幫到入睡
• 對玫瑰花茶有興趣，但唔鍾意茉莉花味
• 媽媽都係陽虛質，諗住買俾媽媽飲
• 6/15 約咗沙田分店覆診

呢份摘要由 Jessica 自動寫，跟住每次對話開頭都會 load 入 context。所以下次 Apple 提起「我媽媽」， Jessica 已經知道係指乜。

中醫師看到嘅效果：呢份 CRM notes 可以直接俾醫師參考，知道病人過往同 Jessica 講咗咩。診症前花 30 秒睇 notes，就掌握咗病人嘅完整 context。

🎯 三個功能加埋一齊嘅效果

Jessica 唔再係「每次對話都從零開始」嘅機器人。佢有短期記憶（rolling window）、長期記憶（auto-summary），仲識主動 surface 出嚟。 對病人感覺：「Jessica 真係記得我」。對中醫師感覺：「呢個 AI 真係識做嘢」。
